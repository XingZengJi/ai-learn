# Building with the Claude API - 56 The server inspector 服务器检查器

> Course: Building with the Claude API · Lesson 56
> 课程: Building with the Claude API · 第 56 课

搭建 MCP 服务器时,你需要一种方式,能在不连接完整应用的情况下测试你的功能。Python MCP SDK 内置了一个基于浏览器的检查器(inspector),让你能实时调试和测试自己的服务器。

## 启动检查器

首先,确保你的 Python 环境已经激活(具体命令请查阅项目的 README)。然后用以下命令启动检查器:

```bash
mcp dev mcp_server.py
```

这会在 6277 端口启动一个开发服务器,并给你一个本地 URL,在浏览器里打开即可。检查器界面会随即加载,显示出 MCP Inspector 的控制台。

## 关于界面的重要说明

MCP 检查器还在持续开发中,所以你看到的界面可能和当前的截图有出入。不过,测试工具、资源和提示词的核心功能应该保持类似。

## 连接并测试工具

点击左侧的「Connect」按钮来启动你的 MCP 服务器。连接成功后,你会看到一个导航栏,里面有 Resources(资源)、Prompts(提示词)、Tools(工具)等板块。

测试你的工具:

1. 进入 Tools 板块
2. 点击「List Tools」,查看所有可用工具
3. 选择一个工具,打开它的测试界面
4. 填入所需的参数
5. 点击「Run Tool」执行,查看结果

## 测试文档操作

比如,要测试文档读取工具,你可以输入一个文档 ID(比如 `deposition.md`),然后运行这个工具。检查器会显示结果,包括返回的内容或成功提示信息。

你可以把多个操作串联起来验证功能。比如,在用「查找替换」编辑完一份文档之后,可以立刻再运行一次「读取」工具,确认改动确实生效了。

## 开发工作流

检查器打造出了一个高效的开发循环:

1. 修改你的 MCP 服务器代码
2. 通过检查器测试单个工具
3. 不需要搭建完整应用,就能验证结果
4. 独立地排查问题

随着你搭建的 MCP 服务器越来越复杂,这个工具会变得不可或缺。它省去了「为了测试基础功能,就要把服务器接到 Claude 或其他应用上」这道工序,让开发速度更快、更专注。

---

对产品经理来说,「服务器检查器」这类工具的价值,和做产品测试时的「沙盒环境」是一个道理:你不需要把一个新功能上线到真实用户面前才能验证它对不对,而是先在隔离的测试环境里跑一遍——省去了「搭完整链路才能测试」的高昂成本,让每一次修改都能快速得到反馈。评估技术团队的开发效率时,「有没有这类独立测试工具」是个值得关注的信号。
