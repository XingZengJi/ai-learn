# Building with the Claude API - 70 Routing workflows 路由工作流

> 课程: Building with the Claude API · 第 70 课

路由工作流(routing workflows)解决的是 AI 应用里一个常见问题:不同类型的用户请求,需要不同的处理方式。与其用一个「一刀切」的提示词,你可以先给传入的请求分类,再把它们路由到各自专门的处理流水线。

## 通用提示词的问题

设想一个能根据用户提供的主题生成视频脚本的社交媒体营销工具。用户可能输入「编程」或「冲浪」作为主题,但这两者应该产出非常不同类型的内容:

编程类主题适合用教育型内容,要有清晰的讲解和定义。冲浪类主题更适合用娱乐向的脚本,强调兴奋感和视觉吸引力。一个通用的提示词没法同时把这两者都处理好。

## 搭建内容分类体系

第一步,是定义你的应用可能需要生成的不同内容类型。你可以把请求归到这样的类别里:

- **娱乐(Entertainment)** —— 高能量、贴合当下文化热点、用语时髦的内容
- **教育(Educational)** —— 清晰、引人入胜的讲解,配上贴近生活的例子
- **喜剧(Comedy)** —— 犀利、出人意料的内容,带着巧妙的观察和节奏感
- **个人 vlog(Personal vlog)** —— 真实、私密的内容,用对话式的叙事方式
- **评测(Reviews)** —— 态度鲜明、基于真实体验的内容,突出优缺点
- **讲故事(Storytelling)** —— 沉浸式的内容,用生动的细节和情感连接吸引人

每个类别都有自己专属的提示词模板。比如,教育类的提示词可能会要求 Claude「打造一份清晰、引人入胜的脚本,用贴近生活的例子和发人深省的问题,把复杂信息转化成易于消化的洞察」。

## 路由实际是如何运作的

路由过程分两步:

1. **分类(Categorization)** —— 把用户的主题发给 Claude,要求它把这个主题归到你预先定义好的某一类里
2. **专门处理(Specialized Processing)** —— 用分类结果来选择对应的提示词模板,生成内容

举个例子,如果用户输入「Python 函数」作为主题,你会先让 Claude 给它分类:

```
Categorize the topic of a video into one of the listed categories:
<topic>Python functions</topic>

<categories>
- Educational
- Entertainment  
- Comedy
- Personal vlog
- Reviews
- Storytelling
</categories>
```

Claude 会回答「Educational(教育)」,于是你接下来就用教育类的提示词模板,来生成实际的脚本内容。

## 路由工作流的架构

一个路由工作流遵循这样的模式:

1. 用户输入首先进入一个「路由器」组件
2. 路由器用一次初始的 Claude 调用,对请求进行分类
3. 根据分类结果,输入被转发到一条特定的处理流水线
4. 每条流水线都可以有自己专属的工作流、提示词或工具,针对该类别进行了优化

核心洞察是:用户输入只会进入「一条」专门的流水线,而不是全部流水线都跑一遍。这让每条流水线都能针对自己的具体场景做高度优化。

## 什么时候该用路由

路由工作流适合以下场景:

- 你的应用要处理种类各异、需要不同处理方式的请求
- 你能清楚地定义出覆盖你所有使用场景的分类体系
- 分类这一步能被 Claude 可靠地完成
- 「专门处理带来的效果提升」大于「路由这一步额外增加的开销」

这种模式对客服机器人、内容生成工具,以及任何「正确回答高度依赖于先理解请求类型」的应用来说,都特别有价值。

---

对产品经理来说,路由工作流对应的是客服体系里常见的「分诊台」设计:用户一进来先简单问一句「您想咨询哪方面的问题」,再把 ta 转接给对应的专业客服组,而不是让每个客服都用同一套万能话术应付所有类型的问题。分类这一步看似多余,但正是它让后面每个环节都能做得更专、更精,这个道理不管用在人工客服还是 AI 系统上都是一样的。
