## 什么是工件?

工件(artifacts)是 Claude 在对话旁边的独立窗口中创建的、可独立存在的交互式输出。你看到的不再是聊天记录里埋着的一长串代码或文字,而是直接渲染好、随时可用的内容——无论是一个可运行的网站、一张交互式图表,还是一份可以立即下载的文档。

当内容满足以下几个条件时,Claude 会自动创建一个工件:

- 内容分量足够、能自成一体,通常超过 15 行
- 这是你很可能想要编辑、迭代或复用的内容
- 内容本身足够复杂,脱离对话上下文也能独立存在
- 这是你以后会想引用或重复使用的内容

## 常见的工件类型

Claude 能创建多种类型的工件,分别适合不同需求:

- **文档(Documents)**(包括 Markdown、纯文本、Word 文档、PDF、PowerPoint 和 Excel):适合任何文字量大、你想导出或继续编辑的内容——比如会议纪要、报告、项目计划、博客文章等书面内容。
- **代码片段(Code snippets)**:任意编程语言的可运行代码——Python、JavaScript、C++ 等等。你可以查看代码、复制代码,或下载后用于自己的项目。
- **HTML 页面(HTML pages)**:包含 HTML、CSS 和 JavaScript 的完整网页,全部打包在一个文件里,非常适合落地页、表单、交互演示或快速原型。
- **SVG 图像(SVG images)**:用于 logo、图标、插画等视觉元素的可缩放矢量图形。这类图形会直接在工件窗口中渲染出来,所以你能准确看到最终效果。
- **Mermaid 图表(Mermaid diagrams)**:流程图、时序图、甘特图、组织架构图等。描述你想可视化的关系,Claude 就会创建一份你可以继续调整的图表。
- **React 组件(React components)**:具备真实功能的交互式 UI 元素——计算器、仪表盘、小游戏、数据可视化。这些不只是效果图,而是包含真实逻辑、能响应用户输入的可运行组件。

## 创建你的第一个工件

创建工件就像正常聊天一样简单。只需描述你想要什么,Claude 会自行判断是否要以工件的形式呈现。

举例来说,你可以这样说:

> "Create a flowchart showing our customer onboarding process (Note: Claude may now generate visual diagrams like flowcharts as HTML using Imagine, in addition to code-based artifacts.)"
> 画一张流程图,展示我们的客户入职流程(注:除了基于代码的工件,Claude 现在也可能通过 Imagine 以 HTML 的形式生成流程图之类的可视化图表)。
>
> "Build an interactive dashboard that lets me input monthly expenses and see a breakdown"
> 做一个交互式仪表盘,让我能输入每月支出并看到明细分类。
>
> "Design a landing page for a productivity app with a hero section and feature list"
> 为一款效率类应用设计一个落地页,包含主视觉区和功能列表。
>
> "Write a project brief template I can reuse for new initiatives"
> 写一份我可以在新项目中反复使用的项目简报模板。

如果 Claude 在你预期会生成工件的地方没有自动创建,你也可以明确要求:"把这个做成工件"("Create this as an artifact")或"用工件展示给我看"("Show me this in an artifact")。

当 Claude 生成一个工件时,它会出现在对话右侧的一个独立窗口中。在这里,你可以:

- **查看不同格式**:在预览效果(呈现出来的样子)和底层代码之间切换
- **复制内容**:点击复制图标,把内容拿去用在别处
- **下载文件**:把工件保存为文件到你的电脑
- **查看代码**:准确看到 Claude 在背后生成了什么

## 分享与发布工件

创建出有用的内容后,你有几种分享方式可以选择。

**复制或下载**:如果是个人使用,或想通过其他渠道分享,可以使用工件窗口右下角的复制或下载按钮。

**在组织内部分享(Claude for Work)**:Team 和 Enterprise 用户可以在组织内部与同事分享工件。共享的工件仅保留在你的组织内,访问需要团队身份验证。

**公开发布**:免费版、Pro 和 Max 用户可以公开发布工件,让任何拿到链接的人都能访问。发布时:

- 只有你选中的那个版本会公开(你的对话本身仍然是私密的)
- 任何人都能查看并与该工件互动,无需 Claude 账号
- 其他人可以对你的工件进行"remix"(二次创作)——在自己的 Claude 对话中打开它,在此基础上修改和继续开发

要发布,点击工件右上角的"Share"或"Publish"按钮。你可以随时回到该工件,取消公开访问来撤下发布。**注意**:一旦发布工件,任何人都能通过链接访问,即使没有 Claude 账号也能查看。已发布的工件不会被搜索引擎收录,所以不会出现在 Google 搜索结果中。

## 用好工件的小技巧

**说清楚你想要什么**。"做一个预算追踪工具"这样说可以,但"做一个月度预算追踪工具,让我能按类别输入支出、看到饼图分类,并在超支时收到提醒"会更好。

**说明最终使用者是谁**。告诉 Claude 谁会使用这个工件,能帮它做出更合适的设计选择。"这张流程图是给新员工看的"和"这张流程图是给工程团队看的",会得到不同的结果。

**逐步迭代**。每次只让 Claude 添加一个功能或做一处改动,这样更容易判断哪里有效、并及早发现问题。

**需要时主动要求生成工件**。如果你要求的内容分量不小,但 Claude 只是在聊天中回复而没有创建工件,直接说一句"请把这个做成工件"即可。
