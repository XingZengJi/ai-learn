## 学习目标 (What you'll learn)

学完本课后,你将能够:

- 识别 Claude 桌面应用中的三种模式——Chat(对话)、Cowork(协作办公)和 Code(编程)——以及每种模式各自的设计目的
- 说明每种模式独有的关键功能,包括快速唤出、定时任务、本地开发与远程开发的区别
- 根据你要完成的工作类型,选择合适的模式

## Claude 桌面应用导览:Chat、Cowork、Code

Claude 桌面应用提供三种与 Claude 协作的方式:Chat、Cowork 和 Code——从快速提问,到复杂研究,再到构建软件,都能覆盖。

Chat 就是你在 claude.ai 上熟悉的那个 Claude,再加上原生运行在电脑上带来的快速唤出、截图、语音输入和连接器等功能。Cowork 是一个智能体(agentic)工具——你给它一个目标,把它接入你的工具和资源,让它自主完成工作。借助 Cowork,Claude 拥有更大的触达范围和操作空间,这种更广的作用域让它能够进行更彻底的研究与分析,产出更复杂的文档与交付物。Code 用于构建软件,从编写、测试代码到部署都能完成。

Cowork 和 Code 运行在同一套引擎上。两者底层都是 Claude Code——运行在你本机上,能够独立作业,可以拉起子智能体(sub-agents)并维持长时间任务。这使得 Claude 能够独立完成较大规模的任务,比如研究撰写或软件构建。

每种模式都是围绕它所服务的工作类型设计的,会把重要信息呈现给你,并在你需要的地方交还控制权。

## Chat

Chat 最适合用来提问、头脑风暴、起草内容,或是来回讨论解决问题。

如果你用过 claude.ai,这里的体验是一样的,再加上几项因为原生运行在电脑上而带来的功能:

- **快速唤出(Quick entry)**:在 Mac 上双击 Option 键,就能在你正在做的任何事情上方唤出 Claude。它会在一个紧凑的窗口中回复,并在你切换应用时始终保持置顶。你完全不需要离开手头的工作就能提问。
- **截图与窗口分享(Screenshots and window sharing)**:截取一张屏幕截图,或分享一个窗口,让 Claude 准确看到你正在看的内容,比口头描述屏幕内容更快、也更精确。(仅 Mac)
- **语音输入(Dictation)**:用说的方式讨论问题,而不是打字。在你需要边想边说、离开键盘,或者说话比打字更快的场景下很有用。(仅 Mac)
- **桌面连接器(Desktop connectors)**:通过连接器接入本地工具和服务,让 Claude 能够与你电脑上的其他工具协同工作。

**适合尝试的场景**:

- 你正盯着一个陌生的仪表盘发愁。双击 Option,拖动光标截取窗口画面,问一句"这些指标是什么意思?"Claude 会在悬浮窗口中作答,同时仪表盘依然留在视野内。
- 你在两场会议之间的空隙,想理一理一份演示文稿该怎么组织。打开快速唤出,切换到语音模式,把想法说出来,Claude 会根据你说的内容起草一份大纲。
- 你几周以来一直在 Apple Notes 里零散地记录一个产品发布的想法。你在设置中添加了 Notes 连接器,并让 Claude:"把我笔记里关于 Osprey 发布的内容整理到一起,找出我哪些地方写到一半没写完,再检查一下我连接的其他工具,看看有没有能补上空缺的信息。"Claude 会读取你本机上的笔记,把已有内容拼接起来,并在你中断的地方续上。

## Cowork

Claude Cowork 是为需要真正花功夫的工作而设计的:从多个信息源中提取信息、理清脉络,并产出一份完整的成果。

在 Cowork 中,Claude 可以同时处理项目的不同部分,因此既有足够的作用范围去调用更多信息源,也有足够的"耐力"把事情做完——比如详尽的研究简报、跨信息源的财务分析、端到端的合同审查,或是把散落在各处的素材整理成一份精美的幻灯片。

在开始之前,Claude 通常会先问几个简短的问题,确认你的需求:范围、格式、限制条件。它会构建一份计划,你可以在侧边栏中查看。在它工作的过程中,你能看到任务逐步成型:它正在参考的信息源、正在成形的文件、计划的完成进度。你可以同时运行多个任务,每个任务都在各自独立的对话中,并可以在侧边栏之间切换。

- **文件夹访问(Folder access)**:把 Claude 指向电脑上的一个文件夹,它会读取里面的内容,判断哪些相关,并把完成的工作保存回同一个位置。你也可以上传文件、把内容粘贴进对话,或连接能为 Claude 提供所需信息的工具。
- **定时任务(Scheduled tasks)**:Claude 可以按计划处理周期性工作:每天从你的 Slack 和日历中提取信息生成简报、每周汇总当周上线的内容、每天早上帮你分拣收件箱、标出需要关注的邮件。你只需定义任务内容和运行时间,只要应用处于打开状态,Claude 就会每次自动执行。如果任务到点时你的电脑或应用是关闭的,等你回来后它会补上。
- **子智能体(Subagents)**:Claude 拉起的后台工作单元,用于并行处理任务的不同部分。如果你提出一个复杂需求——比如需要整合多个信息源的研究简报——Claude 会把它拆解成若干子任务,把每个子任务分配给一个拥有独立上下文的子智能体,并统筹汇总结果,最终交给你一份完整的成果。
- **移动端调度(Dispatch)**:一条持久化的对话线程,让你可以在手机上继续 Cowork 中的对话。通过 Claude 手机应用,你可以把用到电脑上一切资源的任务交给 Claude——你的文件、连接器、插件,甚至桌面应用。使用这项功能需要同时安装桌面版和移动版应用,并保持电脑处于唤醒状态、桌面应用处于打开状态。
- **项目(Projects)**:Cowork 中的项目功能让你把相关任务归入专属工作区,拥有各自的文件、上下文、指令和记忆。如果你在 Claude 上用过项目功能,Cowork 的项目用法类似,只是它们保存在你本机的桌面上,并且是围绕你在 Cowork 中运行的任务构建的。
- **浏览器操作(Browser use)**:在 Chrome 中连接 Claude 后,它可以浏览网站、与页面互动,并把找到的内容直接汇入正在处理的任务。Cowork 正是靠这项能力完成诸如"核对十个网站上的竞品定价"或"从没有 API 的页面上抓取数据"这类工作。
- **电脑操作(Computer use)**:当 Claude 没有对应的连接器或插件可用时,它可以直接操作你的电脑——像你一样点击、输入、打开应用。Claude 会按优先级顺序尝试:优先用连接器,其次是 Chrome,最后才是直接操作屏幕,以确保总是走最快、最可靠的路径。每次 Claude 要访问某个应用前,你都会看到权限提示,你也可以设置屏蔽名单,把不希望它接触的内容排除在外。电脑操作功能目前是 Pro 和 Max 套餐的研究预览版,仅支持 macOS(Windows 版即将推出)。
- **插件(Plugins)**:插件能赋予 Claude 它本身不具备的能力:比如拉取实时财务数据、搜索公司内部知识库,或在特定合规框架下工作。你可以在 Cowork 界面中浏览并添加插件,以匹配你的任务需求。
- **受保护的运行环境(Protected environment)**:Cowork 在你电脑上的一个受限空间内运行。Claude 可以在你共享的文件夹范围内读取、创建和编辑文件,但无法访问这个范围之外的任何内容。

**适合尝试的场景**:

- 你想像查数据库一样查询你所有的工具。问一句:"把上季度我们在会议记录、Slack 和邮件里对定价做出的决定汇总一下,再根据这些发现更新我们的 Q3 演示文稿",Cowork 就会跨会议记录、幻灯片、邮件和 Slack 讨论串找出答案。
- 你在研究一个新市场、梳理竞争对手、评估各种工具。对于任何可能涉及多个浏览器标签页、信息难以提取的研究,Cowork 会自己访问网站、阅读报告、提取定价数据,并给你一份带来源引用的结构化简报——你甚至不用打开一个浏览器标签页。
- 你需要梳理一个包含 50 多份项目文档的文件夹,里面有合同、财务报告和会议记录。你可以让 Cowork 找出与你的项目最相关的文档,并产出一份摘要备忘录。Cowork 会阅读每一页内容,在整套文档间交叉比对,提炼出只有通读全部文档才能发现的规律——审阅 50 份文档就像审阅 5 份一样轻松。
- 你每天早上都要重复做同样的事——查看消息、整理状态更新、为当天的会议做准备。把它设置成一次性的定时任务,Claude 就会每天自动重复执行,让你以"已有答案"而不是"待办杂务"开始新的一天。

Cowork 面向 Pro、Max、Team 和 Enterprise 用户开放,并会定期新增功能。

## Code

Code 标签页让你能够在桌面应用内直接使用 Claude Code 的全部能力,为你提供一个完整的开发环境来构建软件。

通过 Code,Claude 直接在你的代码库中工作:读取现有内容、编写和修改代码、运行命令。可视化差异对比(diff)展示每次改动,内置终端实时显示正在运行的命令,git 记录每一个版本,让你随时可以回滚。

Cowork 运行在一个受限的工作空间中,范围限于你共享的文件夹;而 Code 则直接运行在你的项目里,对你的文件系统、终端和开发工具拥有完整访问权限。

你可以选择工作运行的位置:

- **本地(Local)**:你在电脑上选择一个文件夹,Claude 直接操作这些文件。由于运行在你本机上,Claude 可以读取你的项目、访问本地工具,并运行一个你可以在浏览器中预览的开发服务器。
- **远程(Remote)**:你连接一个 GitHub 仓库,Claude 在云端环境中工作。即使你关闭应用,会话仍会继续,因此你可以启动一次大型重构,之后再回来查看进展。适合较大的代码库,或者你想让开发过程不占用本机资源的情况。

三种交互模式让你可以控制 Claude 自主程度的高低:

- **Ask**:Claude 会提出每一处改动,并等待你的批准。你先审查可视化差异对比,再决定接受或拒绝,任何改动才会真正生效。
- **Code**:Claude 会自动应用文件改动,但在运行终端命令前会先向你确认。
- **Plan**:Claude 会在动手之前先给出完整的方案。一个专门的计划查看器让你可以在工作推进过程中审阅并回顾这份策略。

你可以跨项目同时运行多个会话,并在侧边栏中按状态(进行中 Active / 已归档 Archived)和环境(本地 Local / 云端 Cloud)筛选。

Code 标签页面向 Pro、Max、Team 和 Enterprise 用户开放。

## 三种模式对比 (Comparing the three modes)

| | Chat | Cowork | Code |
|---|---|---|---|
| **适用场景** | 更快节奏的交流:探讨想法、反复起草、快速问答、在对话中学习 | 复杂或持续性的工作:研究、分析、文件整理,产出完整的文档与交付物 | 构建软件:编写、测试、运行和部署代码 |
| **关键功能** | 快速唤出、语音输入 | 从本地文件夹取材、插件、子智能体、定时任务 | Ask / Code / Plan 三种模式、可视化差异对比、git 集成、本地与远程环境 |
| **工具与扩展** | 连接器、Skills(技能)、Claude in Chrome | 连接器(本地与远程)、Skills、Claude in Chrome、插件、电脑操作(Computer Use) | 连接器、Skills、Claude in Chrome、插件、Hooks(钩子) |
