## 一、全课程知识点清单

### 第 1 课 什么是 Claude

- **宪法式 AI(Constitutional AI)**:Claude 的设计三原则——乐于助人(helpful)、无害(harmless)、诚实(honest)
- **不只是聊天机器人**:摘要、搜索、创意/协作写作、问答、编程都擅长,定位是「思维伙伴」
- **可引导性**:能按指示调整个性、语气和行为
- **上下文窗口**:20 万+ token(约 500 页文本);付费套餐用受支持模型最高 100 万 token
- **混合模型两种模式**:近乎即时的响应 + 扩展思考(extended thinking,逐步推演)
- **学习模式(Learning mode)**:引导你推理而非直接给答案
- **访问方式**:网页/桌面/移动应用,登录后对话、项目、记忆、偏好跨设备同步,所有套餐可用
- **Claude 是智能本身**,多种界面通向同一个智能:Claude.ai、Claude Code、@Claude(Slack)、Claude Design、Claude for Microsoft 365

### 第 2 课 与 Claude 的首次对话

- **像同事一样交流**:自然、简洁、富有对话感
- **好提示词三要素**:①设定场景(角色/目标/背景) ②明确任务(要它做什么) ③指定规则(风格/语气/示例)——源自 4D 框架的「描述」能力
- **上传文件加上下文**:支持 PDF、DOCX、CSV、TXT、PNG、JPEG;能同时分析文本与视觉元素
- **迭代三招**:追问细节 / 给具体反馈 / 引导回正题(最坏开新对话刷新上下文);铅笔图标可编辑重发提示词
- **个性化两功能**:Memory(自动记住角色、偏好、决策,可在设置中查看/编辑/删除,跨设备同步)、Styles(预设或自定义交流风格,全局生效)

### 第 3 课 获得更好的效果

- **五大常见问题与解法**:
  - 太笼统 → 补受众/角色/限制条件的细节
  - 太长或太短 → 明确说篇幅要求
  - 格式不对 → 给格式范例,「展示而非仅描述」
  - 一本正经地说错(幻觉)→ 高风险工作独立核实、让它注明来源、开网络搜索
  - 语气不对 → 直白描述想要的语气,附风格范例
- **迭代式思维**:初稿是起点;反馈要具体;跑偏太远时果断开新对话
- **AI 流畅度 4D 框架**:委派 Delegation(人做什么 AI 做什么)、描述 Description(如何有效沟通)、辨别 Discernment(批判性评估输出)、勤勉 Diligence(负责任、合乎道德地用)
- **轻量评估(evals)四步**:收集 5–10 个自己做过的样本 → 写测试提示词 → 对比 Claude 输出与样本 → 据此优化提示词/确定人工审核环节

### 第 4 课 桌面应用三模式:Chat / Cowork / Code

- **Chat**:快节奏交流。独有:快速唤出(Mac 双击 Option)、截图/窗口分享、语音输入、桌面连接器
- **Cowork**:复杂持续性工作(研究简报、跨源分析、文件整理)。关键能力:文件夹访问、定时任务、子智能体(subagents,并行处理并汇总)、移动端 Dispatch、项目、浏览器操作、电脑操作(优先级:连接器 > Chrome > 直接操作屏幕)、插件、受保护的运行环境(只能访问你共享的文件夹)
- **Code**:构建软件,对文件系统/终端/开发工具完整访问。本地(Local)vs 远程(Remote,云端,关应用也继续跑);三种自主度:Ask(每处改动等批准)/ Code(自动改文件,跑命令前确认)/ Plan(先出完整方案)
- **底层关系**:Cowork 和 Code 运行在同一套引擎上,底层都是 Claude Code
- Cowork 和 Code 面向 Pro/Max/Team/Enterprise 用户

### 第 5 课 Introduction to Projects(项目)

- **定义**:自成一体的工作区,拥有各自的记忆、对话历史、知识库和自定义指令——为特定工作流打造的专属环境
- **知识库(Project knowledge)**:上传的文档会被该 Project 内所有对话引用,不用每次重新上传
- **指令(Instructions)**:设定语气、专业程度、回复风格等,适用于该 Project 内的每一次对话;也可用来自动化流程(如"上传会议记录就按模板生成摘要")
- **自动扩容(RAG)**:知识库接近上下文容量上限时,Claude 无缝启用检索增强生成(Retrieval Augmented Generation)模式,容量最多扩展 10 倍,同时保持回复质量(**官方测验第 5 题考点**)
- **团队协作(Claude for Work 用户)**:三种权限——可查看(Can view,只读+可讨论)、可编辑(Can edit,可改指令/知识/成员)、所有者(Owner,决定谁能看到该 Project)
- **创建步骤**:命名 + 描述 → 写指令(背景/流程/语气/具体要求)→ 建知识库(上传文档,文件名要具体,如 "Q4-2024-Brand-Guidelines.pdf" 而非 "document1.pdf")
- **典型应用**:产品发布知识库、研究支持、客户账号信息中心、活动策划工作区、职位描述生成器
- **最佳实践**:先聚焦一个具体场景再扩展、定期更新知识库、指令要具体、文件命名规范、提问时按文件名引用以聚焦搜索范围

### 第 6 课 Artifacts(工件)

- **定义**:对话旁独立窗口中、可独立存在的交互式输出
- **自动创建四条件**:分量足(通常 >15 行)、你可能会编辑/迭代/复用、脱离对话也自成一体、以后会引用
- **六种类型**:文档、代码片段、HTML 页面、SVG 图像、Mermaid 图表、React 组件
- **窗口内操作**:预览/代码切换、复制、下载、查看源码
- **分享方式**:复制下载(个人用)/ 组织内分享(Team、Enterprise)/ 公开发布(Free、Pro、Max)
- **公开发布规则**:只公开选中的版本,对话仍私密;任何人凭链接可访问无需账号;他人可 remix;**不会被搜索引擎收录**;可随时撤下
- **技巧**:说清楚要什么、说明给谁用、逐步迭代、需要时明确说「做成工件」

### 第 7 课 Skills(技能)

- **定义**:指令+脚本+资源构成的文件夹,Claude 动态加载,像「专业能力包」/ SOP
- **两类**:Anthropic 内置(Excel/Word/PPT/PDF 文档创建,付费用户自动可用)、自定义(固化你的工作流程)
- **启用前提**:先开「代码执行与文件创建」,因为 Skills 依赖沙盒环境;Enterprise 要 Owner 先在管理端启用
- **自动触发**:场景相关时 Claude 自动调用,不需手动指定
- **文件处理**:Chat 模式下生成的是文档新版本,不是原地编辑原文件
- **安全**:只装可信来源的自定义 Skill;自定义 Skill 仅对个人账号私有
- **创建方式**:直接和 Claude 对话即可,无需写代码;在「Customize」标签页管理
- **Projects vs Skills**:Projects 存放知识(是什么),Skills 执行任务(怎么做);二者互补,Skill 可引用 Project 里的知识

### 第 8 课 连接器(Connectors)

- **作用**:让 Claude 访问你日常工具的数据,既能读取也能代你执行操作
- **底层是 MCP(Model Context Protocol)**:「AI 的 USB-C 接口」,开放的通用连接标准
- **两类**:网页连接器(Google Drive、Notion、Slack 等云服务)、桌面扩展(本机运行,访问本地文件和原生应用,需桌面应用)
- **入口**:claude.ai/directory 目录,或对话框「+ > Connectors」
- **安全三点**:权限范围受限且可单独开关;**Claude 只能看到你本人有权限看到的**;随时可撤销连接

### 第 9 课 企业搜索(Enterprise Search)

- **形态**:侧边栏专属「Ask {组织名}」Project,相当于为全组织预搭好的知识库 Project,用 Anthropic 配置的检索专用指令
- **仅 Team 和 Enterprise 套餐**,需管理员启用
- **两步设置**:先由组织所有者(Owner)完成初始配置(选文档类+聊天类连接器),再由每位用户自行完成各服务的身份验证
- **典型问题**:了解近况、政策流程、跨源研究、新人入职、项目追踪;回答总附来源
- **安全**:只显示你在原始工具中本就有权限的内容;对话私密;数据不被单独索引或存储

### 第 10 课 Research 模式

- **本质**:智能体式多步骤调查——连续多次相互衔接的搜索,自主决定下一步查什么
- **耗时**:多数 5–15 分钟,复杂的最长 45 分钟
- **联动**:启用 Research 会**自动开启扩展思考**;**前提是已启用网络搜索**
- **每条论述都带引用来源**,便于核实
- **四工具选择**:快速事实 → 网络搜索;纯推理(数学/调代码/逻辑)→ 扩展思考;组织内部知识 → Enterprise Search;多源综合调查 → Research
- **提示词技巧**:明确目标、给出想要的结构章节、给限制条件(预算/时间线/地点)、可先让 Claude 帮你打磨提示词
- **进阶**:可关掉网络搜索只查已连接工具,做纯内部研究

### 第 11 课 按角色的应用案例

- 通用:状态报告、用户反馈分析、品牌规范打包成 Skill
- 销售(battle card、洽谈准备)、市场(投放分析、跨平台改写)、财务(建模、投资备忘录、读表格)、HR(入职指南)、法务(证据开示时间线)、研究(文献综述、核对统计)
- 更多灵感:claude.com 用例展示库(Use Case Gallery)

### 第 12 课 Claude 的更多使用方式

- **Claude Code**:终端/IDE/浏览器里的智能体编码工具——建功能、调试、读陌生代码库、自动化琐事
- **@Claude**:Slack 内标记即用——总结讨论串、会议准备、从 bug 讨论直接派编码任务
- **Claude Design**:文字/草图/截图 → 交互式 UI 原型,可对接团队设计系统
- **Claude for Excel / PowerPoint / Word / Outlook**:M365 侧边栏原地分析、起草、编辑,上下文跨应用延续(Outlook 版是独立安装的测试版)
- **Claude in Chrome**:浏览器侧边栏,能观察页面并直接操作——总结网页、填表自动化、测试网站流程。**公开测试中:建议只在可信网站做低风险任务**,高风险操作(购买、分享个人数据)前会请求许可,金融/成人类网站默认屏蔽
