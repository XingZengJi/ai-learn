# Claude 101 - 12 Other Ways to Work with Claude Claude 的更多使用方式

> 课程: Claude 101 · 第四章「综合运用」第 2 课(全课程第 12 课)
> 预计用时: 10 分钟

## 学习目标

学完本课后,你将能够:

- 了解何时使用 Claude 的其他产品,包括 Claude Code、@Claude、Claude Design、Claude for Microsoft 365 和 Claude in Chrome

正如本课程开始时提到的,Claude 是一套智能本身,Claude.ai 只是与它协作的其中一种方式。

Claude 还内置在多款专门的工具中,让你可以在自己已经在用的地方直接使用它。本课将介绍与 Claude 协作的更多方式,每一种都针对特定的工作流程和使用场景。

## Claude Code

Claude Code 是一款智能体编码工具,可以在你工作的地方直接运行——终端、IDE、浏览器,甚至 Slack 里都可以。它能理解你的代码库、执行命令,并通过自然语言处理完整的开发工作流程。

**什么时候用 Claude Code**:

- 你想用大白话描述需求来构建功能,让 Claude 编写代码、运行测试、创建提交
- 你需要调试问题——把报错信息贴给 Claude,让它分析你的代码库,定位并修复问题
- 你在浏览一个不熟悉的代码库,想问问不同部分是如何协同工作的
- 你想自动化一些繁琐的工作,比如修复 lint 报错、解决合并冲突,或撰写发布说明
- 比起切换到一个独立界面,你更喜欢在终端里搭配现有的 IDE 和开发工具工作

## @Claude

@Claude 把 Claude 直接带入 Slack,让你能在频道和讨论串中获得帮助,或者把 Slack 中的上下文带入你与 Claude 的对话——只需在任意讨论串中标记 @Claude 即可。

**什么时候用 @Claude**:

- 你想起草消息回复、总结冗长的讨论串,或理清复杂的讨论内容,而不用离开 Slack
- 你需要准备会议,让 Claude 从你的工作区中整合相关对话和共享文档
- 你刚加入一个新团队,想通过回顾频道历史记录来了解正在进行的项目
- 你想直接从一条 bug 报告或功能讨论中把编码任务交出去——只需标记 @Claude,它就能结合上下文启动一个 Claude Code 会话
- 你在讨论过程中需要快速了解行业趋势、技术概念或公司信息

## Claude Design

Claude Design 是一个专门用来把创意转化为可用界面的空间。用大白话描述你想要的效果——或者从一张草图、参考截图开始——Claude 会构建一个交互式原型,你可以通过对话不断完善,再交付给团队。

**什么时候用 Claude Design**:

- 你想从一份文字简报、草图或参考截图,不写代码就得到一个可用的 UI 原型
- 你在探索不同的设计方向,想快速生成并比较多个版本
- 你想通过描述你想要的改动来迭代布局、文案或交互,而不是手动修改标记语言
- 你希望原型使用你团队的设计系统,这样交付的东西能和工程团队最终实现的一致

## Claude for Excel

Claude for Excel 以侧边栏的形式把 Claude 直接带入 Microsoft Excel,让你可以通过对话分析、理解并修改电子表格。

**什么时候用 Claude for Excel**:

- 你在处理一个复杂的多标签工作簿,想了解特定公式或跨工作表的计算流程是如何运作的
- 你需要更新模型中的假设或输入值,同时保留公式的依赖关系和相互联系
- 你在调试类似 #REF!、#VALUE! 或循环引用这样的表格错误,想让 Claude 追溯到根源并给出修复建议
- 你想创建新的电子表格,或用数据填充现有模板,同时保持正确的公式结构
- 你需要快速创建数据透视表或图表,把数据可视化呈现出来

## Claude for PowerPoint

Claude for PowerPoint 以侧边栏的形式把 Claude 带入 Microsoft PowerPoint,让你可以通过对话起草、编辑和重组演示文稿,同时保留你原有的模板和品牌样式。

**什么时候用 Claude for PowerPoint**:

- 你想把一份大纲、文档或笔记转换成初稿幻灯片,而不用逐页手动搭建
- 你需要重写或精简幻灯片文案——缩短要点、添加演讲者备注,或针对特定受众调整语气
- 你在重组一份现有的演示文稿,想调整章节顺序、拆分信息过密的幻灯片,或合并内容重叠的部分
- 你想让整份演示文稿的格式保持一致——标题、要点样式、版式——而不用逐页手动调整
- 你想为某一页幻灯片快速获得视觉呈现建议,比如哪种版式或图表类型最适合表达你的观点

## Claude for Word

Claude for Word 以侧边栏的形式把 Claude 带入 Microsoft Word,让你可以起草、修订、重组当前打开的文档——支持修订模式和批注,并能从已连接的信息源中提取背景,为你写的内容提供依据。

**什么时候用 Claude for Word**:

- 你想把一份大纲或粗略的笔记,按团队模板整理成结构完整的初稿
- 你需要修订某一部分——精简文字、针对特定读者调整语气,或调整结构——而不用离开文档
- 你在处理审阅者的批注和修订记录,想直接在文档里逐条处理
- 你希望草稿的内容有依据可循,基于你已连接的源材料,让文档中的论述都能追溯到出处

## Claude for Outlook

Claude for Outlook 以侧边栏的形式把 Claude 带入你的收件箱,让你可以分拣邮件、结合相关讨论串和日历信息起草回复,并把一长串邮件往来整理成清晰的摘要或行动清单。

> 可用性:Claude for Outlook 目前处于测试版阶段,与其他 Microsoft 365 插件是分开安装的。安装步骤请参见 Claude for Outlook 相关说明。

## Claude in Chrome

Claude in Chrome 是一款浏览器扩展,会在 Google Chrome 中以侧边栏形式加入 Claude。它能观察你正在做的事情,并直接在浏览器内执行操作。

**什么时候用 Claude in Chrome**:

- 你想在浏览时总结文章、研究论文或网页内容
- 你需要帮忙起草邮件回复或管理收件箱
- 你在填写重复性的表单,想把这个过程自动化
- 你想测试网站功能,或走完一个多步骤流程,而不用手动逐步点击
- 你需要一个在你切换标签页和任务时能保持上下文的浏览助手——这让它特别适合从小众的内部工具、CRM 或仪表盘中提取信息

> 重要提示:Claude in Chrome 目前处于公开测试阶段。Anthropic 建议只在可信网站上用它执行低风险任务。这个扩展在执行诸如购买或分享个人数据这类高风险操作前会请求你的许可,某些类别的网站(金融服务、成人内容)默认会被屏蔽。

## 小结

以上每一款工具,都是把 Claude 的能力延伸到你实际工作的具体环境中:

| 工具 | 最适合 | 运行位置 |
|---|---|---|
| Claude.ai | 通用任务、研究、写作、分析、文件创建 | 网页、桌面和移动应用 |
| Claude Code | 软件开发、代码库浏览、git 工作流 | 终端/命令行、IDE 或浏览器 |
| Claude Cowork | 复杂的多步骤任务:研究简报、文档创建、文件整理、数据分析 | 桌面应用(通过 Dispatch 也可用移动应用) |
| @Claude | 团队协作、会议准备、结合上下文的快速问答 | Slack 工作区 |
| Claude Design | UI 原型、设计探索、基于设计系统的原型稿 | 网页 |
| Claude for Microsoft 365 | 原地编辑,并在多个文档间延续上下文 | Excel、PowerPoint、Word、Outlook 侧边栏 |
| Claude in Chrome | 网络研究、邮件管理、浏览器自动化 | Chrome 浏览器侧边栏 |
