## 课程结构

第一部分是一条**以「语境文档」为主线**串起来的实践链:

1. **打底**(01–03)—— 合作背景 → AI Fluency 定义 → 4D 框架,**并做出语境文档(从价值观写起)**
2. **认识机器**(04–05)—— 交互式行为指标自查 + 生成式 AI 的能力与局限
3. **两个环各练一遍**(06–07)—— 第 06 课练描述–辨别(做分层教学材料),第 07 课练委派–审慎(学生数据 + 伦理承诺)
4. **收束成个人立场**(08)—— 把语境文档整理成**意图 / 原则 / 边界**三段式的 AI 价值观文档
5. **合起来 + 收尾**(09–10)—— 端到端跑一遍完整 4D;最后从「完成任务」升级到「设计一个东西解决问题」

**那份语境文档贯穿全程**: 第 03 课创建(以价值观开头)→ 第 07 课加入伦理承诺 → 第 08 课整理成 AI 价值观文档。**三次迭代,是这门课的隐藏主线。**

## 几条值得单独记住的

**4D 直接对应教师已有的技能**(03)—— **描述 ≈ 备课**(定义产出、步骤、成功标准),**辨别 ≈ 评估学生作业与课程材料**。**不是学新技能,是把既有技能用在新协作者身上。**

**让 AI 反过来采访你**(03)—— 建语境文档时,不要自己描述自己,**让它提问**。因为你不知道自己遗漏了什么;你描述只会说出已经想到的。

**24 条行为指标**(04)—— 把四项抽象能力拆成「你做到了没有」的可观察行为。其中最容易被忽略的两条: **告诉 AI 你希望它怎么和你互动**(当思考伙伴,不当应声虫)、**察觉你和 AI 是否在共同推进**(比察觉答案错误更难,也更省时间)。

**修改提示词要「动作 + 对象 + 具体标准」**(06)—— **不是「做得更好一点」**,而是「把说明的步骤缩短,并给每一步加图示提示」。**「更好一点」把判断标准原封不动推回给了 AI,而它没有那个标准。**

**贴样本胜过描述风格**(06)—— 上传过去的单元计划、评分量表、优秀范例,比写十条格式要求管用。

**辨别检查清单的最后一条**(06)—— **「有没有东西会在你这批具体的学生身上翻车?」** 前几条问内容对不对,这条问放到具体人群里合不合适——**这一条 AI 完全无法自查,它不认识你的学生。**

**三句伦理填空**(07)—— ①我不会委派的决策是 ___ ②AI 有用是当它帮我或学生 ___ ③不确定时我会 ___。**第三条最重要,因为灰色地带才是常态。**

**FERPA 同样适用于 AI 工具**(07)—— 普适版本: **你现有的数据合规义务,不会因为换了个工具而消失。**

**AI 不是教学法,你才是**(08)—— 抵抗「把工具当成方法论」这个误解。

**意图 / 原则 / 边界**(08)—— 三段式对应「我想达成什么 / 我打算怎么用 / 哪些我不会用它做」。**大多数人只有模糊的第一项;第三项最有价值也最难写,因为它要求你放弃一些明明可行的便利。**

**拥有过程,而不是拥有产出**(09)—— 所以练习要的是**成品 + 一份怎么做出来的记录**,后者才是训练目标。配套一问: **「我补上的、只有我才能提供的东西是什么?」——答不上来,说明这事本就该完全自动化。**

**解决问题 ≠ 完成任务**(10)—— 任务场景里描述和辨别最吃重,**问题场景里委派和审慎才是难点**,因为没人告诉你产出该是什么。

**四段式工作坊结构**(ttt-03)—— 热身反思 / 核心活动 / 委派矩阵 / 收尾承诺。**首尾最容易被砍、也最不该砍**: 没有热身内容就是外部信息,没有承诺培训结束即失效。

## 与其他 AI Fluency 课程的关系

本仓库同系列课程:

- `Course/Fluency/` —— **教育者版(通用)**
- `Course/FluencyforStu/` —— 学生版
- `Course/14T-AIFluency/` —— Teaching AI Fluency(如何把 4D 教给别人)
- `Course/15AIFAIfornon/` —— 公益组织版
- `Course/19AIF4SB/` —— 小微企业版
- `Course/20AIforB/` —— 建造者版
- `Course/20AIforPK12/`(本课)—— **pK–12 教育者版 + 培训者培训**

**与建造者版(20AIforB)的对照**最能看出这套框架怎么换语境:

| | 建造者版 | pK–12 版(本课) |
|---|---|---|
| 主线练习 | **一个项目贯穿五课**(诊所候诊查询器) | **一份语境文档贯穿三课**(价值观 → 伦理承诺 → 价值观文档) |
| 起点 | 问题简报(用户、需求、约束) | **价值观**(你希望学生成为什么样的人) |
| 辨别的对象 | 代码与 UX,分**五个镜头** | 教学材料的**准确性、难度、目标对齐、人群适配** |
| 伦理篇幅 | 并入审慎一课 | **单设一课(07),外加价值观文档一课(08)** |
| 独有工具 | 描述链、五个镜头、三种协作模式 | **24 条行为指标、意图/原则/边界三段式** |
| 有无带教包 | 无 | **有(第二部分完整工具包)** |

**最大的差别在起点**: 建造者版从**问题**出发,pK–12 版从**价值观**出发。这不是风格差异——**教育场景里,「学生本该自己完成的思考被替代了吗」这类问题无法用效率衡量**,所以框架必须先锚定价值观才能往下走。

另外,`Course/14T-AIFluency/`(Teaching AI Fluency)与本课第二部分主题相近,但定位不同: **前者讲「怎么教 4D」的方法,后者直接给你一套 pK–12 场景的现成材料。**
