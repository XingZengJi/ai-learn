## 课程主线

1. **概念**(01)—— 子智能体在**独立上下文窗口**里运行,只把摘要交回主线程,中间过程全部丢弃。核心价值是**保护主上下文这个有限资源**,代价是失去过程可见性。内置三个: General purpose、Explore、Plan
2. **创建**(02)—— 用 `/agents` 命令;选作用域(项目级 / 用户级)、工具权限、模型(Haiku/Sonnet/Opus/Inherit)、颜色;配置文件落在 `.claude/agents/*.md`,frontmatter + 系统提示词
3. **设计**(03)—— 四条要点: 具体的描述、**定义输出格式**、障碍汇报、限制工具权限
4. **选型**(04)—— 一条决策规则加三个反模式

## 最值得记住的几条

**描述有双重作用**(第 3 课):它既决定主智能体**何时**启动这个子智能体,又被当作写作指引来生成**发给子智能体的输入提示词**。所以描述写得含糊,子智能体收到的任务也会含糊。

**定义输出格式是单条改进里效果最大的**(第 3 课):它给了子智能体自然的停止点。没有它,子智能体判断不出"研究到什么程度算够",会跑得远超必要时长。

**「遇到的障碍」应当是输出模板的固定一节**(第 3 课):子智能体发现的变通办法、环境怪癖、需要特殊参数的命令,如果不显式要求汇报,就会随任务结束而丢失,主线程得重新踩一遍。

**决策规则**(第 4 课):**中间过程重要吗?** 不重要就委派,重要就留在主线程。

**三个反模式**(第 4 课):

- **「专家」人设** —— 「你是 X 专家」不增加任何能力,Claude 本来就有那些知识
- **顺序流水线** —— 每步依赖上一步发现时,信息会在交接中丢失(修 bug 几乎总是如此)
- **测试运行器** —— 会隐藏你调试需要的完整输出;课程明确说这个模式在所有配置中表现最差
