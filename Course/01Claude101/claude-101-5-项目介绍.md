## 学习目标

学完本课后,你将能够:

- 解释什么是 Projects(项目),以及什么时候该用它
- 创建一个新 Project,设置名称、描述和可见性
- 把文档和文件添加到 Project 的知识库中
- 撰写有效的 Project 指令来引导 Claude 的行为
- 与队友分享 Project(面向 Claude for Work 的 Team 和 Enterprise 套餐用户)

## 关键要点

- Projects 是自成一体的工作区,拥有各自的记忆、对话历史、知识库和自定义指令。可以把它们理解成为特定工作流打造的专属环境。
- Project 知识库能增强 Claude 的理解——你上传相关文档后,Claude 会在该 Project 内的所有对话中引用这些内容,不用每次都重新上传同样的文件。
- Project 指令能引导 Claude 的行为——你可以指定语气、专业程度、回复风格等等。这些指令适用于该 Project 内的每一次对话。
- Projects 会自动扩容。当知识库接近上下文容量上限时,Claude 会无缝启用检索增强生成(Retrieval Augmented Generation,RAG)模式,把容量最多扩展 10 倍,同时保持回复质量。
- 对 Claude for Work 用户而言,Projects 支持协作。把 Project 分享给队友,大家就能共享同一套背景信息、指令和积累下来的知识。

## 什么是 Projects?

Projects 特别适合三件事:存放 Claude 应该参考的知识、围绕特定主题或工作领域整理相关对话,以及和需要访问同一套共享背景信息的团队成员协作。

## 什么时候该用 Projects

当你在做的是一件持续性的事情——而不只是一次性的提问时,Projects 尤其有价值。当你的工作流程符合以下情况时,可以考虑创建一个 Project:

- 有会反复用到的参考材料(会议记录、调研结果、报告、历史数据等)
- 对 Claude 该如何回复有一致的要求(总是用正式语言、总是注明来源、总是遵循我们的模板)
- 需要团队协作,多个人要基于同一套基础信息工作

## 创建你的第一个 Project

设置一个 Project 只需要几分钟。步骤如下:

**第一步:创建 Project**

1. 把鼠标悬停在左侧边栏,点击「Projects」,或直接访问 claude.ai/projects
2. 点击右上角的「+ New Project」
3. 给 Project 起一个描述性的名字(比如"Q4 营销活动"或"产品文档")
4. 添加一段简短描述,说明你在做什么。虽然 Claude 不会直接看到这段描述,但它能帮你和队友理解这个 Project 的用途
5. 选择可见性设置:保持私密,或与你的组织共享(面向 Claude for Work 用户)

**第二步:添加 Project 指令**

Project 指令告诉 Claude 在这个 Project 内的所有对话中该如何表现。点击「Instructions」打开指令面板。

好的 Project 指令通常包括:

- 关于你在做什么的背景信息:"这个 Project 用于为我们的 B2B 软件产品创作营销内容。"
- 流程指令:"先构思一个能吸引这类受众的博客结构,再写初稿。"
- 语气和风格偏好:"使用专业但不失亲切的语气,尽量避免行话。"
- 具体要求:"营销文案结尾始终要包含一个行动号召(call-to-action)。"

写好指令后,点击「Save instructions」保存。这些指令会应用到该 Project 内的每一次对话,并和你设置的用户偏好、风格一起生效。

你也可以用 Project 指令来自动化工作流程——例如:"当我上传一份会议记录,就用这个模板生成一份结构化摘要。"可以把指令理解成为这个 Project 给 Claude"编程"设定行为模式。

**第三步:搭建知识库**

Project 的知识库是你上传 Claude 应该参考的文档的地方。在 Project 主页面右侧能找到文件菜单。

点击「+」按钮添加内容。你可以上传多种文件类型,包括 PDF、DOCX、CSV、TXT、HTML 等等,也可以连接 Google Drive 直接关联文档。

**该上传什么**:

- 参考文档(品牌规范、风格指南、模板)
- 背景材料(研究报告、会议记录、需求文档)
- 你希望 Claude 模仿的作品范例
- 技术文档或规格说明

> 专业提示:给文件起一个有描述性的名字。Claude 会用文件名来理解和检索正确的信息,所以 "Q4-2024-Brand-Guidelines.pdf" 比 "document1.pdf" 更有用。

## Projects 如何处理大型知识库

你可能会好奇,上传大量内容后会发生什么。Projects 会通过一个叫检索增强生成(Retrieval Augmented Generation,RAG)的过程自动扩容,来处理海量内容。简单来说,这意味着 Claude 能在回答问题时自动找到并使用你上传文档中最相关的部分,而不需要你告诉它该看哪个文件。

当你的 Project 知识库接近上下文窗口上限时,Claude 会无缝启用 RAG 模式。它不会把所有 Project 内容一次性加载进内存,而是智能地搜索并只检索回答你问题所需的最相关信息。这能把你 Project 的容量最多扩展 10 倍,同时保持回复质量。

当你的 Project 启用 RAG 模式时,你会看到一个视觉提示,但使用体验应该是一样的——你依然可以上传文档、和 Claude 对话,获得贴合上下文的回复。

## 在 Project 中工作

Project 设置好之后,你就可以开始和 Claude 对话了。该 Project 内的每一次对话都会自动访问你的知识库,并遵循你的 Project 指令。

## 协作功能

对于 Claude for Work(Team 和 Enterprise)套餐用户,Projects 通过协作功能会变得更加强大。

**权限级别**

分享 Project 时,你可以选择三种权限级别:

- **可查看(Can view)**:成员可以看到 Project 内容、访问知识库并对话——但不能做出改动。可以理解成带讨论权限的只读访问。
- **可编辑(Can edit)**:成员拥有完整的协作权限。他们可以修改指令、更新知识库、管理其他成员,并积极为该 Project 做贡献。
- **所有者(Owner)**:Project 创建者掌控一切,包括谁能看到这个 Project。他们可以分享给特定的人,也可以让整个组织都能看到这个 Project。

**分享你的 Project**

分享 Project 的步骤:

1. 打开你想分享的 Project
2. 点击 Project 名称右侧的「Share project」按钮
3. 用姓名或邮箱添加单个成员,或者复制粘贴一份邮箱地址列表进行批量分享(这种情况下,该 Project 会出现在对方的"Shared with you"分区)
4. 或者,分享给"[你的组织] 的所有人",让这个 Project 能在 Team 标签页中被发现

当你分享一个 Project 给团队成员时,他们会收到邮件通知,并且可以在自己的「Shared with me」标签页中找到被分享的 Project。

## 一些能给你启发的 Project 示例

不确定该从哪里开始?以下是不同职能中常见的一些 Project 类型:

- **Q4 产品发布**:上传产品规格说明、竞品分析,以及消息传递的头脑风暴笔记。无论是回答问题还是起草文档,Claude 都会随时把这些背景信息放在心上。
- **研究支持**:把竞品评审、用户研究数据和客户反馈集中存放。Claude 能帮你整合信息源、起草报告,并在各项建议之间保持一致性。
- **客户账号信息中心**:把客户的品牌规范、过往交付物和沟通记录集中放在一个地方。设置好指令后,Claude 在制作提案或报告时,就能匹配他们的语气,并参考他们的具体背景。
- **活动策划工作区**:上传场地合同、演讲嘉宾简介和参会者数据。Claude 能帮你生成活动流程单、参会者通知,以及和活动主题保持一致的活动后报告。
- **职位描述生成器**:收集过往的职位描述、团队章程和内部编制申请文档。和 Claude 一起打磨出真正反映你团队实际工作和文化的职位描述。

## Projects 使用最佳实践

想要充分利用 Projects,可以参考:

- **先聚焦,再扩展**。从一个具体的使用场景开始,而不是试图创建一个囊括一切的 Project。你随时可以在过程中添加更多内容。
- **保持知识库是最新的**。过时的文档会导致过时的回复。定期审查并更新你的 Project 知识库。
- **写清晰的指令**。具体说明你想要什么。模糊的指令会导致结果不一致。
- **给文档起有描述性的名字**(比如 "Q4-2025-Sales-Report.pdf" 而不是 "report.pdf"),并把相关文件归类放在一起。Claude 会利用文件名和文件之间的关联,来理解文档之间的关系。
- **按名称引用文档**。提问时,可以提到具体的文档名称,帮助 Claude 聚焦搜索范围:"根据我们的 Q3 报告,客户最关心的问题排名前几位的是什么?"

## 课程反思

在进入下一课之前,不妨思考:

- 你手头哪些持续性的工作,可能因为有一个带持久化背景信息的专属 Project 而受益?
- 你预计自己会经常向 Claude 重复上传或重复解释哪些文档?
- 如果你在一个团队里,有没有哪些 Project 会因为共享的知识库和指令而受益?
