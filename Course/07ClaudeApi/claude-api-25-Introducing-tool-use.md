# Building with the Claude API - 25 Introducing tool use 工具使用入门

> Course: Building with the Claude API · Lesson 25
> 课程: Building with the Claude API · 第 25 课
> 来源: Anthropic Academy (anthropic.skilljar.com),用户登录后提供英文原文,2026-07-17 整理翻译存档

工具(tools)让 Claude 能够获取外部世界的信息,把它的能力扩展到训练数据之外。默认情况下,Claude 只知道训练数据里的信息,没办法获取时事、实时数据或外部系统的信息。「工具使用」通过建立一套结构化的方式,让 Claude 能够请求并接收新鲜信息,解决了这个限制。

## 没有工具时的问题

当用户向 Claude 询问当下的实时信息时,Claude 会直接撞墙。比如,如果有人问「旧金山现在天气怎么样?」,Claude 只能回答类似「抱歉,我没有获取实时天气信息的能力」这样的话。

这会造成很糟糕的用户体验——明明 Claude 理论上是可以帮上忙的,只要它能拿到当下的实时数据就行。

## 工具使用是如何运作的

工具使用遵循你的应用程序和 Claude 之间一套特定的往返模式。完整流程如下:

1. **初始请求**:你把问题连同「如何从外部来源获取额外数据」的指令一起发给 Claude
2. **工具请求**:Claude 分析问题,判断自己需要额外信息,于是明确请求它需要什么数据
3. **数据获取**:你的服务器运行代码,从外部 API 或数据库中取回所请求的信息
4. **最终回答**:你把取回的数据发回给 Claude,Claude 结合原始问题和这份新鲜数据,生成完整的回答

## 天气示例实操

我们用天气这个问题来看看具体怎么运作,整个过程会变得非常具体:

当用户询问当前天气时,你在提示词里加入「如何获取天气数据」的说明。Claude 意识到自己需要实时信息,于是针对具体地点请求天气数据。你的服务器随即调用天气 API 获取实时状况,再把这份数据发回给 Claude。最后,Claude 把新鲜的天气数据和用户的问题结合起来,给出准确、及时的回答。

## 核心优势

- **实时信息**:能获取 Claude 训练时还不存在的当下数据
- **对接外部系统**:能连接数据库、API 和其他服务
- **动态回答**:能基于最新的可用信息给出答案
- **结构化交互**:Claude 清楚知道自己需要什么信息、该如何请求

工具使用把 Claude 从一个「静态知识库」变成了一个能处理实时数据的动态助手。这为构建各种需要当下信息的应用打开了可能性——无论是天气数据、股票价格、数据库查询,还是用户可能需要的任何其他实时信息。

---

对产品经理来说,「工具使用」就是给 Claude 接上了「外部数据接口」:Claude 本身像一个知识渊博但「与世隔绝」的顾问,只懂培训时学过的东西;工具则相当于给这个顾问配了一部能随时打电话查最新行情的手机——它自己判断什么时候该打这个电话、该问什么,你的系统负责真正拨号接通并把结果转告给它。
