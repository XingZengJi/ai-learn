# Building with the Claude API - 26 Project overview 项目总览

> Course: Building with the Claude API · Lesson 26
> 课程: Building with the Claude API · 第 26 课

接下来我们要做一个实战项目,教会 Claude 为未来的日期设置提醒。这件事乍一看很简单,但深入下去会发现几个有意思的挑战,我们要用自定义工具来一一解决。

目标很直接:我们希望能对 Claude 说「帮我设个提醒,提醒我去看医生,时间是下周四」,然后 Claude 回答「好的,我会提醒你」。但要让这件事真正运转起来,我们需要解决 Claude 在处理时间和提醒这件事上的一些局限。

## 为什么这件事有挑战

虽然 Claude 知道当前日期,但有三个具体问题需要解决:

1. **对时间的感知有限**:Claude 可能知道当前日期,但不一定知道精确的时间
2. **日期计算容易出错**:Claude 在处理「基于时间做加法」这类计算时表现并不总是可靠,尤其是要往未来推算好多天的时候
3. **没有设置提醒的能力**:Claude 并不知道该如何「设置一条提醒」——它本身没有任何内置机制能做到这件事

这三个局限,分别代表着「Claude 天生能做的事」和「我们这个提醒系统需要它做的事」之间的一道道缺口。工具,就是用来填补这些缺口的东西。

## 我们需要的工具

我们会创建三个独立的工具,分别解决上面三个挑战:

1. **获取当前日期时间**:Claude 需要精确知道当前的日期和时间
2. **给日期时间加上一段时长**:Claude 在日期时间加法上不够完美,所以我们要给它一个可靠的计算工具
3. **设置提醒**:我们需要一种方式,能在系统里真正地设置一条提醒

我们会按顺序一个个实现这些工具,从最简单的开始。这种方式能让我们先理解「工具调用」到底是怎么运作的,再去搭建更复杂的功能。到项目完成时,Claude 就能处理「一周后提醒我」这类自然语言请求了——它会组合调用这几个工具,先算出精确的时间点,再设置提醒。

这个项目演示了和 AI 打交道时的一条关键原则:当模型有局限时,我们应该用工具去扩展它的能力,而不是在提示词里绕着这些局限打转、想办法「绕过去」。

---

对产品经理来说,这一课其实是在示范「能力缺口分析」的标准做法:先列清楚系统现在做不到什么(不知道精确时间、算日期不可靠、不能真的落地设提醒),再针对每一条缺口配一个具体的解决方案(工具),而不是笼统地要求「让 AI 更聪明点」。把一个模糊的大需求(设置提醒)拆解成几个可以独立验证的小能力,是产品设计里非常常见、也非常好用的思路。
