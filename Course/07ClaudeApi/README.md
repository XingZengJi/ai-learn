# Building with the Claude API(用 Claude API 开发)

Anthropic Academy 官方课程 <https://anthropic.skilljar.com/building-with-the-claude-api> 的中文笔记。

- 命名格式: `claude-api-N-English-Title.md`,`N` 与站点课次顺序一致(从 1 起)。**本课文件名用英文标题**,与仓库里其他课程的中文文件名不同
- 共 **73 课**,是本仓库体量最大的一门课
- 对应成就地图课程 ID: **`c11`**

## 课程结构

73 课分成九大块,是一条从「发出第一个请求」到「搭出智能体」的完整链路:

| 课次 | 主题 | 内容 |
|---|---|---|
| 1–11 | **API 基础** | 访问 API、拿密钥、发请求、多轮对话、系统提示词、温度参数、流式响应、结构化数据(含 3 个练习) |
| 12–18 | **提示词评估(Evals)** | 典型评估工作流、生成测试数据集、运行评估、**基于模型的评分 vs 基于代码的评分** |
| 19–24 | **提示词工程** | 清晰直接、表达具体、用 XML 标签组织结构、提供示例(含练习) |
| 25–36 | **工具使用(Tool Use)** | 工具函数与 Schema、处理消息块、发送执行结果、多轮与多工具、精细化调用,以及两个内置工具: **文本编辑工具**与**网络搜索工具** |
| 37–43 | **RAG 检索增强生成** | 文本分块策略、嵌入、完整 RAG 流程与实现、**BM25 词法搜索**、多索引流水线 |
| 44–47 | **模型能力** | 扩展思考、图像支持、PDF 支持、引用溯源 |
| 48–51 | **成本与执行** | 提示词缓存(概念/规则/实战三课)、代码执行与 Files API |
| 52–62 | **MCP** | 从零实现: 定义工具、服务器检查器、实现客户端、定义与访问资源、定义提示词、复习 |
| 63–73 | **智能体与工作流** | Anthropic 出品的应用、Claude Code 搭建与实战、用 MCP 增强、**四种工作流模式**(并行化/链式/路由)、智能体与工具、环境检视,以及收尾的**工作流 vs 智能体** |

**几个值得注意的编排**:

- **评估(12–18)排在提示词工程(19–24)之前** —— 先学怎么衡量好坏,再学怎么改进。这个顺序不是偶然
- **提示词缓存单独占了三课**(48–50): 概念、规则、实战。**「规则」单独一课**说明缓存的命中条件容易踩坑
- **最后一课「工作流 vs 智能体」是全课的落点**,前面十课铺的都是它

## 与其他课程的关系

- `Course/12ClaudewithAB/`(**c14**)—— **Claude with AWS Bedrock,与本课高度重合**。两门课的 1–61 课几乎一一对应(基础 → 评估 → 提示词工程 → 工具 → RAG → 模型能力 → 缓存 → MCP),差别在于本课直接调 Anthropic API,那门课走 AWS Bedrock。**已经学过一门,另一门可只看差异部分**
- 第 52–62 课的 MCP 部分与 `Course/11MCP/`(**c09**,MCP 进阶)互补: 本课教你**从零实现**,那门课讲**协议进阶能力与传输层**
- 第 64–66 课的 Claude Code 部分与 `Course/02ClaudeCode101/`(**c02**)重合,后者更完整
