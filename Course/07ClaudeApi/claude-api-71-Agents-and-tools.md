# Building with the Claude API - 71 Agents and tools 智能体与工具

> Course: Building with the Claude API · Lesson 71
> 课程: Building with the Claude API · 第 71 课
> 来源: Anthropic Academy (anthropic.skilljar.com),用户登录后提供英文原文,2026-07-17 整理翻译存档

智能体(agents)代表着一种和我们之前所做的「结构化工作流」不同的转变。工作流适合「你清楚知道完成任务需要哪些确切步骤」的场景;而智能体的价值,在于「你不确定这些步骤该是什么样」的时候。你不需要定义一套僵化的执行顺序,而是给 Claude 一个目标和一套工具,让它自己想办法组合这些工具来达成目标。

这种灵活性,让智能体在搭建那些「需要处理各种各样、难以预测的任务」的应用时格外有吸引力。你可以只搭建一次智能体,确认它表现得还不错,然后就能用它去解决一大批不同的问题。不过,这种灵活性也伴随着可靠性和成本上的取舍,我们之后会具体探讨。

## 工具是如何成就智能体的

智能体真正的威力,在于它能以出人意料的方式把简单的工具组合起来。来看一套基础的日期时间工具:

- `get_current_datetime` —— 获取当前日期和时间
- `add_duration_to_datetime` —— 给指定日期加上一段时长
- `set_reminder` —— 为特定时间创建提醒

这些工具单独看起来都很简单,但 Claude 能把它们串联起来,处理出人意料复杂的请求:

对于「现在几点?」,Claude 只需要调用 `get_current_datetime`。但对于「11 天后是星期几?」,它会先调用 `get_current_datetime`,再调用 `add_duration_to_datetime`。要在「下周三」设一个健身房提醒,它可能会依次用上全部三个工具。

Claude 甚至能识别出自己什么时候需要更多信息。如果你问「我的 90 天保修期什么时候到期?」,它会知道要先问你「你是什么时候买的这件商品」,才能算出到期日。

## 工具应该是抽象的

搭建有效智能体的关键洞察是:提供相对抽象的工具,而不是高度专门化的工具。Claude Code 完美地展示了这个原则。

Claude Code 能访问的是通用、灵活的工具,比如:

- `bash` —— 运行任意命令
- `read` —— 读取任意文件
- `write` —— 创建任意文件
- `edit` —— 修改文件
- `glob` —— 查找文件
- `grep` —— 搜索文件内容

值得注意的是,它并没有「重构代码」或「安装依赖」这类专门化的工具。相反,Claude 会自己想办法用这些基础工具来完成这些复杂任务。正是这种抽象化,让它能处理无数种开发者从未明确规划过的编程场景。

## 最佳实践:可组合的工具

设计智能体时,要提供 Claude 能以创造性方式组合的工具。比如,一个做社交媒体视频的智能体可能会包括:

- `bash` —— 访问 FFMPEG 来处理视频
- `generate_image` —— 根据提示词生成图像
- `text_to_speech` —— 把文字转换成语音
- `post_media` —— 把内容上传到社交平台

这套工具集,既能支撑简单的工作流(创建并发布一条视频),也能支撑更具互动性的体验——比如智能体可能先生成一张示例图片、等用户批准,再继续制作视频。

智能体能根据用户的反馈和偏好调整自己的做法,这是僵化的工作流很难做到的。正是这种灵活性,让智能体在搭建动态的、能响应用户需求的应用时特别有力。

---

对产品经理来说,「工具应该抽象」这条原则值得反复琢磨:很多团队第一反应是「给 AI 助手做个『重构代码』的专用按钮」,但 Claude Code 的成功恰恰证明了相反的思路——给它「读文件」「写文件」「跑命令」这类基础、通用的能力,让它自己组合出解决方案,反而能覆盖远比你事先能想到的更多场景。这和「授人以渔」的道理一样:与其为每个具体需求单独定制一个工具,不如给一套灵活的基础能力,让智能体自己去拼装解法。
