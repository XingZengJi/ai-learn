# Building with the Claude API - 73 Workflows vs agents 工作流 vs 智能体

> Course: Building with the Claude API · Lesson 73
> 课程: Building with the Claude API · 第 73 课

搭建 AI 驱动的应用时,你经常需要在两种不同的架构方式之间做选择:工作流(workflows)和智能体(agents)。两者各有明显的优势和取舍,适合不同的场景。

## 什么是工作流?

工作流是一系列预先定义好的、对 Claude 的调用,用来解决一个已知的问题或一类问题。当你能提前想清楚步骤流程时——也就是你确切知道完成任务需要哪个顺序——就该用工作流。

可以把工作流理解成:把一个大任务拆解成更小、更具体的子任务。每一步都专注于单一的领域,这让 Claude 能更精准地完成工作。

## 什么是智能体?

用智能体的时候,Claude 拿到的是一套基础工具,并被期望自己制定计划,用这些工具来完成任务。和工作流不同,你并不确切知道会遇到什么样的任务,所以整个系统需要具备更强的适应性。

智能体能通过以出人意料的方式组合工具,创造性地应对各种各样的挑战。

## 工作流的优势

- Claude 可以一次专注于一个子任务,通常带来更高的准确率
- 更容易评估和测试,因为你确切知道每一个步骤
- 执行结果更可预测、更可靠
- 更适合解决具体、边界清晰的问题

## 智能体的优势

- 能带来更灵活的用户体验
- 完成任务的方式远比工作流灵活——Claude 能以出人意料的方式组合工具,应对各种各样的任务
- 能处理开发过程中没有预料到的新奇情况
- 需要时能主动向用户询问更多信息

## 工作流的缺点

- 灵活性差得多——只专注于解决特定类型的任务
- 用户体验通常更受限——你需要提前知道这个流程确切的输入是什么
- 需要更多的前期规划和设计工作

## 智能体的缺点

- 相比工作流,任务成功完成的比率更低
- 更难以埋点、测试和评估,因为你往往不知道智能体会执行怎样的一连串步骤
- 行为的可预测性更差

## 什么时候该用哪种方式

作为工程师,你的首要目标是可靠地解决问题。用户大概率不关心你搭建了多么「花哨」的智能体——他们想要的是一个能稳定工作的产品。

总体的建议是:只要可能,就始终优先实现工作流,只有在真正需要的时候才诉诸智能体。工作流能提供大多数生产级应用所需要的可靠性和可预测性,而智能体则为那些「确切需求没法提前定死」的场景,提供了灵活性。

当你面对的是边界清晰的流程时,考虑用工作流;当你需要处理不可预测、多变、需要创造性解决问题的用户请求时,再考虑用智能体。

---

对产品经理来说,这一课其实是在纠正一个常见的误区:「用了 agent 听起来更先进、更有卖点」,但从用户体验和工程可靠性的角度看,能用工作流解决的问题,用工作流往往才是更负责任的选择——用户要的是「每次都靠谱」,而不是「技术架构听起来多炫」。做技术选型决策时,应该先问「这个任务的步骤我们能不能提前想清楚」,而不是先问「要不要上 agent」。
