# Building with the Claude API - 21 Being specific 表达要具体

> Course: Building with the Claude API · Lesson 21
> 课程: Building with the Claude API · 第 21 课

和 Claude 打交道时,提升效果最有效的方法之一,就是把「你想要什么」说具体。与其把一切都交给模型自己去猜,不如给出清晰的指引或步骤,把 Claude 引导到你想要的那种输出上。

可以这样理解:如果你让 Claude「写一个短篇故事,主角发现了自己隐藏的天赋」,Claude 可以往无数个方向发挥——故事可能是 200 字,也可能是 2000 字;可能只有一个角色,也可能有五个;「天赋被发现」这件事本身也可以是任何形式的场景。

加上具体的指引后,你就给了 Claude 一个更清晰的目标。这能显著提升输出的一致性和质量。

## 两类指引

在提示词里表达具体性,主要有两种方式,专业场景里往往会把两者结合使用。

### 输出质量指引

第一类指引,聚焦于列出你想要的输出应该具备哪些特质。这类指引能帮你控制:

- 回答的长度
- 结构和格式
- 需要包含的具体属性或元素
- 语气或风格要求

比如,你可以要求故事字数控制在 1000 字以内、要有一个能揭示主角天赋的明确情节动作、至少要有一个配角。

### 过程步骤

第二类指引,是给 Claude 提供具体的执行步骤。当你希望 Claude 系统性地思考一个问题、或在给出最终答案前考虑多个角度时,这种方式尤其有用。

与其让 Claude 直接动笔写,你可以要求它:

1. 先构思三个能制造戏剧张力的天赋
2. 从中挑一个最有意思的
3. 勾勒出揭示这个天赋的关键场景
4. 构思能强化效果的配角类型

## 实际效果

具体性带来的差异是巨大的。在一次饮食计划提示词的测试中,加上具体指引后,评估分数从 3.92 提升到了 7.86——仅仅是告诉 Claude 要包含哪些具体要素,输出质量就翻了一倍还多。

指引示例:

1. 准确列出每日卡路里总量
2. 标明蛋白质、脂肪、碳水的摄入量
3. 注明每餐的具体进食时间
4. 只使用符合饮食限制的食材
5. 所有分量都以克为单位列出
6. 如果提到了预算,要控制在预算内

## 两种方式各自适用的场景

### 输出指引:几乎每次都该用

绝大多数提示词都应该带上质量指引。它们是你获得一致、可用结果的「安全网」。

### 过程步骤:用于复杂问题

以下情况适合加上分步骤的指令:

- 排查复杂问题
- 需要做决策判断的场景
- 需要批判性思考的任务
- 任何你希望 Claude 从多个角度考虑问题的情况

举个例子,如果你让 Claude 分析「为什么销售团队的业绩下滑了」,你会希望它依次检查市场指标、行业变化、个人表现、组织变动、客户反馈——而不是让它自己随便挑一个原因就下结论。

### 两者结合使用

在专业场景的提示词工程中,你经常会看到这两种技巧一起用:既有指引控制输出的格式和内容,又有步骤确保 Claude 在给出结论前把问题想透彻。

这种组合既能保证结果的一致性,也能让你确信 Claude 已经把所有重要因素都考虑到了。

---

对产品经理来说,这一课说的其实是写需求文档时最基本的两件事:一是「验收标准」(输出要有哪些字段、多长、什么格式),二是「操作步骤」(遇到复杂判断时,要求执行方按流程一步步来,而不是凭感觉一步到位)。少了前者,团队会用差不多的标准糊弄过关;少了后者,复杂问题容易被简化成单一维度的草率结论。
