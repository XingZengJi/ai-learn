## 课程结构

两大块,**「讲一课 + 练一课」交替推进**:

1. **协议的三项进阶能力**(1–6)—— 每项都是「概念课 + 实操演练」成对出现:
   - **Sampling 采样**(1–2)—— 让**服务器**反过来请求客户端调用模型
   - **日志与进度通知**(3–4)—— 长任务如何向客户端回报进展
   - **Roots 根目录授权**(5–6)—— 客户端告诉服务器它能访问哪些目录
2. **消息与传输层**(7–11)—— JSON 消息类型,然后是两种传输方式: **STDIO**(本地进程)与 **StreamableHTTP**(远程),后者占了三课,含深入解析与**状态管理**

**这个结构本身说明了本课的定位**: 前半讲「协议还能做什么」,后半讲「消息实际怎么传」——都是搭出能跑的 Demo 之后才会碰到的问题。
