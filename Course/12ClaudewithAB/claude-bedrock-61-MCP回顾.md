## Lesson Content 课程内容

MCP 服务器做完了,来回顾三个核心原语(primitives),搞清楚各自该在什么时候用。关键洞察是: **每个原语由技术栈中的不同角色控制**。

## Tools: Model-Controlled 工具: 模型控制

工具完全由 Claude 控制。模型自己决定何时调用这些函数,结果也直接由 Claude 拿去完成任务。

当你想给 Claude **增加能力**时用工具。比如你让 Claude 用 JavaScript 算 3 的平方根,它会自动决定调用一个 JavaScript 执行工具来给出答案。

这个「要用工具」的决定是 100% 由模型作出的——Claude 自己意识到需要执行代码,并挑了合适的工具,应用和用户都没有干预。

## Resources: App-Controlled 资源: 应用控制

资源由你的应用代码控制。什么时候取资源数据、取来干什么,都由应用决定,通常用于界面展示或给对话补充上下文。

当你需要**把数据取进自己的应用**时用资源。常见例子:

- 填充界面上的自动补全选项
- 拉取文档以显示在文件选择器里
- 在把消息发给 Claude 之前补充上下文

Claude 网页版里的「从 Google Drive 添加」就是典型: 应用先拉一份可用文档列表展示在界面上,再把用户选中的那份文档内容注入到对话上下文中。

## Prompts: User-Controlled 提示词: 用户控制

提示词由用户控制。用户通过点按钮、选菜单、敲斜杠命令等直接动作,决定何时触发这些预设工作流。

当你想给用户提供**易于取用的预设工作流**时用提示词。Claude 界面里聊天输入框下方的那排工作流按钮就是这样,让用户快速开始写作、学习、编码等常见任务。

## Choosing the Right Primitive 如何选择

搭建 MCP 服务器时,想清楚**每块功能应该由谁来控制**:

| 你的需求 | 该用 |
|---|---|
| 扩展 Claude 的能力 | 工具 Tools |
| 把数据取进应用界面 | 资源 Resources |
| 给用户提供预设工作流 | 提示词 Prompts |

这是帮你按具体场景做选择的高层原则。三者服务于技术栈的不同层次: **工具服务于模型,资源服务于应用,提示词服务于用户。**

对产品经理来说: 这张「谁来控制」的表是这一整段 MCP 内容里最该记住的东西。它不是技术分类,而是**决策权分配**——你在设计功能时,实际上是在决定「这件事该由 AI 自主判断、由产品逻辑决定,还是交给用户来点」。这个判断做错了,体验就会出问题: 该用户控制的交给模型自主,会显得不可预测; 该模型自主的做成按钮,会让用户觉得繁琐。
