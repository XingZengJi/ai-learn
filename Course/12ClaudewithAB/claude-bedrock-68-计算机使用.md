## Lesson Content 课程内容

Computer Use(计算机使用)让 Claude 能直接与桌面环境交互,本质上是给了它像人一样操作电脑的能力。这为自动化、测试以及各类超出纯文本生成的复杂工作流打开了新的可能。

## What Computer Use Can Do 它能做什么

Claude 不再只是描述该怎么做或生成代码,而是可以真正动手完成任务:

- 截屏以看清屏幕上有什么
- 点击按钮和链接
- 在表单和应用里输入文本
- 在不同应用和浏览器标签页之间切换
- 执行需要视觉反馈的多步流程

这让它在质量保证测试这类工作上尤其有价值——你需要与界面交互,并确认一切如预期工作。

## Real-World Example: Automated QA Testing 实例: 自动化 QA 测试

设想你做了一个带自动补全的 React 组件,用户可以输入 `@` 来提及文件或资源。乍看没问题,但你想把边界情况都测一遍。

与其自己手动点每个场景,不如让 Claude 用 Computer Use 来跑 QA。你给它具体的测试用例:

- 确认输入 "Did you read @" 会弹出自动补全选项
- 测试按回车能正确把一条提及加进文本框
- 检查添加提及后按退格键时,自动补全列表出现的位置是否正确

Claude 会逐条系统地执行,截屏、与界面交互、记录发生了什么。在这个例子里,Claude 发现前两条测试通过,**第三条失败了**——用户按退格键时,自动补全下拉框出现在了错误的位置。

## How the Testing Process Works 测试过程

给 Claude 一个测试任务后,它会按结构化的方式推进:

1. 打开浏览器,导航到你的应用
2. 逐步执行每条测试用例
3. 截屏以验证视觉表现
4. 每条测试之间刷新页面,确保状态干净
5. 记录结果,写清什么通过了、什么失败了
6. 给出带可执行结论的总结报告

关键优势在于: Claude 能抓到你手动测试时可能漏掉的问题,而且每次代码改动后都能一致地重跑同一批测试。

## Setting Up Computer Use 环境配置

出于安全考虑,Computer Use 运行在隔离环境中。典型配置包括:

- 一个运行桌面环境的 Docker 容器
- 一个 Claude 可以控制的浏览器实例
- 一个你用来下达指令的聊天界面
- 与你主系统的完全隔离

这种隔离至关重要——它意味着 Claude 与各类应用和网站交互时,不会给你的个人数据和系统安全带来风险。

## Best Practices 最佳实践

- 明确说明你要 Claude 测什么、完成什么
- 为每个任务给出清晰的成功标准
- 把复杂流程拆成更小、更可控的步骤
- 始终在隔离环境中运行 Computer Use
- 审阅 Claude 的结论,重要结果自己再验证一遍

Computer Use 代表着 AI 能力的显著进步: 从「生成关于任务的文字」走向「真正执行任务」。无论是做 QA 测试、自动化重复流程,还是探索复杂应用,它都能在给出一致、有记录的结果的同时节省大量时间。

对产品经理来说: 这一课最实际的用途是**回归测试**。人工点一遍完整测试用例既枯燥又容易走神漏项,所以现实中往往只在大版本前做一次。而这套东西的边际成本极低,可以每次改动后都跑一遍。注意最后那条最佳实践——「重要结果自己再验证」: 它现在的定位是替你**扩大覆盖面**,不是替你**承担判断责任**。
