## Lesson Content 课程内容

看过 Claude Code 和 Computer Use 之后,可以总结出让智能体(agent)成功的几条关键规律。两者都体现了一种系统化的解题方式: 聚焦的工具使用、对环境的感知、以及迭代式执行。

## How Agents Work in Practice 智能体实际是怎么工作的

当 Claude 被要求为某个边界情况补一个测试时,它的工具调用呈现出清晰的模式: 一共四次调用——两次读已有文件,一次改文件,一次跑测试。

这个拆解透露出一件重要的事: **四次调用里有三次纯粹是在从环境中收集信息**(先搞清楚现有代码库),只有一次真正修改了环境。这种「先观察、再动手」的模式在 Claude Code 和 Computer Use 中都反复出现。

Computer Use 测试网页应用时也是同样的路子: 每次工具调用都返回一张截图,给 Claude 即时的视觉反馈。这个持续的反馈回路让它能理解当前状况,并据此调整下一步动作。

## Comparing Agent Approaches 两者的共同点

Claude Code 和 Computer Use 共享几项让它们有效的根本特征:

- **基于工具的执行** —— 都大量使用工具,并在循环中反复执行,直到达成成功标准或遇到错误
- **环境化的上下文** —— 不依赖详尽的提示词或 RAG 流程,而是**通过工具交互直接获取**上下文
- **聚焦的工具集** —— 每个智能体的工具数量都不多,且各自用途明确
- **高价值、低风险的任务** —— 都针对复杂问题,但犯错的后果是可控的
- **低出错代价** —— Computer Use 出错代价极低,Claude Code 的代价虽不可忽略但也不高

## Key Qualities of Effective Agents 有效智能体的四项关键特质

### 1. Focused Tool Sets Running in Loops 聚焦的工具集,循环运行

智能体在工具数量少、定义清晰简单时表现最好。它们持续执行这些工具,直到触及迭代上限、遇到错误,或满足成功标准。这种迭代方式给了纠偏和改进的空间。

### 2. Context is Everything 上下文就是一切

Claude 对你的具体环境没有任何先天认知。它需要工具去读取和理解目标系统的当前状态。**上下文收集的质量,直接决定智能体的表现。**

### 3. High Value Tasks with Low Error Costs 高价值、低出错代价的任务

智能体擅长复杂的、知识密集型的工作,并且是那种「犯错不会造成重大损害」的工作。写代码就是完美例子: 它需要相当的专业能力,但错误可以被发现和修正,不会带来灾难性后果。

**要避免**把智能体用在高风险决策上——那些一旦出错会造成严重经济或安全影响的场景。

### 4. Continuous Evaluation 持续评估

打造有效智能体的唯一可靠方法就是严格测试。建立评估标准,拿真实场景持续检验智能体的表现。这个反馈回路是发现弱点、提升可靠性的关键。

理解这些规律,也就理解了 Claude Code 和 Computer Use 为什么好用——它们正是围绕这些有效智能体架构的基本原则设计的。自己做智能体时,把这四条记在心里,才能做出既强大又可靠的系统。

对产品经理来说: 第 3 条是选型时最该先问的一条。判断某个场景适不适合上智能体,别先问「AI 做不做得到」,先问「**做错了会怎样**」。写代码错了有测试和代码审查兜底; 但自动发合同、自动打款、自动给客户回承诺——这些错一次的代价,足以抵消它省下的所有时间。这条判断的顺序不能颠倒。
