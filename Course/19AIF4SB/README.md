## 课程结构

九节课是一条串起来的实践链,不是彼此独立的专题:

1. **打底**(01–02)—— 先做一份可复用的**业务语境文档**(课程称为「AI 简报表」),再过一遍 4D。这门课用**内环(描述&辨别,管日常)/ 外环(委派&审慎,管决策)** 的说法,比单纯罗列四个 D 更好懂
2. **认识机器**(03–04)—— 生成式 AI 的原理、能力与局限;交互页动手体验下一词预测
3. **两个环各练一遍**(05–06)—— 第 5 课练描述–辨别(法规追踪、市场调研),第 6 课练委派–审慎(数据脱敏、隐私、透明)
4. **合起来用**(07)—— 全课最长的一节,搭一套完整的自动化工作流并做审慎规划
5. **组织层面**(08–09)—— 起草一页纸 AI 使用政策,以及结课行动清单

**练习前后串联**: 第 1 课的语境文档会在第 2、8 课被反复调用;第 6 课的脱敏数据集贯穿三个连续练习。

## 几条值得单独记住的

**分档依据是后果,不是技术难度**(07)—— 三档分别是「AI 可以处理 / AI 协助人来定夺 / 应由人处理」,判断依据是出错的代价与情绪浓度。**技术可行性完全不在判断依据里。**

**表现描述里最容易漏的三个问题**(07)—— 什么时候升级给人?不确定时怎么办?什么绝对不能做?前两个对应同一类故障:**系统不知道自己不知道**。

**先自己看数据、写下观察,再交给 AI**(06)—— 这个顺序同时给了你校准的尺子、可见的增量价值、和暴露出的盲区。反过来做,你永远分不清哪些洞察是你的。

**「我能解释清楚 AI 在做什么吗?」**(08)—— 把"AI 依赖"这个模糊焦虑变成可回答的问题。不能解释就把流程返工到能解释,而不是干脆不用。

**监管在快速变化**(09)—— 课程特别提醒美国 FTC 正在制定 AI 生成营销内容/评论/客户沟通的指引,各州隐私法也在变。**把披露要求当成需要定期复查的变量。**

## 与其他 AI Fluency 课程的关系

本仓库同系列课程:

- `Course/Fluency/` —— 教育者版
- `Course/FluencyforStu/` —— 学生版
- `Course/14T-AIFluency/` —— Teaching AI Fluency(如何把 4D 教给别人)
- `Course/15AIFAIfornon/` —— 公益组织版(**c19**)
- `Course/19AIF4SB/`(本课)—— 小微企业版(**c20**)

**与公益版(15AIFAIfornon)的对照**最值得一提: 两门课结构高度相似(都是语境文档 → 4D → 两个环 → 合起来 → 组织政策 → 收尾),差别在场景与侧重:

| | 公益版 | 小微企业版 |
|---|---|---|
| 独立的隐私课 | 有(第 5 课单设) | 无,并入第 6 课 |
| 讲 AI 原理 | 无 | **有(第 3–4 课),含交互页** |
| 案例形式 | 虚构人物(Maria、James、Rio、Emily) | **真实企业主出镜**(MAKS Enterprises 的 CEO Mak) |
| 那把尺子 | 效率提升 → 所服务社群的更大受益 | 效率提升 → 更好的客户服务与高价值时间 |
| 合规提醒 | 无 | **有(FTC 指引、州隐私法)** |

另外本课第 3 课的内容,是 `Course/18AICandL/`(**c13**,AI Capabilities and Limitations)的浓缩版——想深入可直接看那门课。
