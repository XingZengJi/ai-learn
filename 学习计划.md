# Anthropic 官方课程学习计划（v2）

> 🌌 **[CC 学习成就地图](https://xingzengji.github.io/ai-learn/)** — 学习进度的星空可视化：90 个知识点是星星，20 门课是星座，5 个项目是行星，学完即点亮。
>
> 制定日期：2026-07-10（v2 修订：技术课程升级为正式学习内容）
> 学习者背景：互联网产品经理出身，非技术背景，已在本机使用 Claude Code
> **学习目标：能独立完成整个工程项目**——因此程序员向的内容也要学，只是要用非技术人员能听懂的方式来学
> 课程来源：https://anthropic.skilljar.com/（Anthropic 官方免费课程，完成可拿证书）
> 学习方式：Skilljar 上看课 + 在本机 Claude Code 里动手实践，两条腿走路

## 一、课程全景图

Skilljar 上共约 20 门课，按学习优先级分三层（v2 调整：原"了解即可"的技术课全部升级）：

### 🟢 第一梯队：使用与协作（6 门）

| 课程 | 一句话说明 | 类比 |
|---|---|---|
| Claude 101 | Claude 的基本用法和核心功能 | 新手机开箱教程 |
| AI Fluency: Framework & Foundations | 与 AI 高效协作的方法论（4D 框架） | 「向上管理」培训——管理对象是 AI |
| Claude Code 101 | Claude Code 完整入门：工作流、上下文、定制化 | 学会带一个全能助理 |
| Introduction to Claude Cowork | 让 Claude 直接操作你的文件和应用 | 助理从「口头汇报」升级到「直接上手干活」 |
| Introduction to Agent Skills | 把重复的指令写成可复用的「技能」 | 给助理写 SOP |
| Introduction to Subagents | 让 Claude 把任务分派给多个「分身」并行处理 | 项目经理带外包团队 |

### 🔵 第二梯队：工程与开发（5 门）— 正式学，用 PM 语言拆解

| 课程 | 一句话说明 | 为什么要学 |
|---|---|---|
| Claude Code in Action | Claude Code 实战：在真实项目中的完整应用 | 从"会用功能"到"能做项目"的桥梁 |
| Introduction to MCP | 给 Claude 接外部工具的「通用接口标准」，含 Python 实作 | 独立做项目时迟早要接数据库、浏览器等外部能力 |
| MCP: Advanced Topics | MCP 的生产环境部署与进阶模式 | 理解"能跑的 Demo"和"能上线的产品"差在哪 |
| Claude Platform 101 | 从零开始在 Claude 开发者平台上构建 | 理解"用 API 做产品"的完整链路 |
| Building with the Claude API | 用 Claude API 开发的全谱系课程 | 做自己的 AI 产品的核心技能 |

### ⚪ 第三梯队：按需选修

- **Claude with Amazon Bedrock / Google Vertex AI**：只在企业采购/云平台选型时才需要，暂缓。
- **AI Capabilities and Limitations**：与 AI Fluency 重叠，当补充读物。
- **AI Fluency for Builders**：面向产品人/创业者，轻量，穿插着看。
- **AI Fluency 其余人群定制版**（Educators / Students / Nonprofits / Small Businesses / Teaching）：同一方法论的定制版，学完主课后无需重复。

## 三、工程基础加油包（随学随补，不单独占周）

独立完成工程项目，除了课程本身，还需要一批"程序员默认你已经会"的基础概念。我们不专门开课，而是**在课程进行到需要它的那一刻，用 15–30 分钟现场讲透**：

| 概念 | PM 版一句话 | 在哪个阶段自然遇到 |
|---|---|---|
| 终端/命令行 | 用打字代替点鼠标来指挥电脑，效率工具而非黑魔法 | 第 3 周装环境时 |
| Git / 版本控制 | 文档的「历史版本 + 多人协作 + 一键回滚」系统，类似飞书文档的版本历史但强大得多 | 第 3 周 Commit 环节 |
| 前端 / 后端 | 前端是用户看得见的界面，后端是柜台后面的厨房 | 第 6 周实战项目 |
| API | 两个软件之间的「服务窗口」：按格式提交申请单，按格式返回结果 | 第 7 周 MCP、第 9 周 API 课 |
| JSON | 软件之间传数据用的标准表格格式 | 第 7 周 |
| Python 基础 | 能"读懂"代码在做什么即可，写由 Claude 代劳 | 第 7–8 周 MCP 实作 |
| 数据库 | 结构化的超级 Excel，软件的记忆系统 | 毕业项目（如需要） |
| 部署/上线 | 把作品从"我电脑上能跑"变成"别人也能访问" | 毕业项目 |

原则：**Claude 写代码，你读懂代码；Claude 执行命令，你明白每条命令在干什么。** 判断学会的标准是你能向别人解释清楚，而不是能默写代码。

## 四、分阶段学习计划（约 10 周，每周 3–5 小时）

### 第 1 周：打地基 —— Claude 101 + AI 协作方法论（上）

**上课**：
- [ ] Claude 101 全部 4 章（Meet Claude → 组织工作 → 扩展能力 → 综合应用）
- [ ] AI Fluency 前半（4D 框架：Delegation 委派、Description 描述、Discernment 判断、Diligence 尽责）

**本机实践**：
- [ ] 实践 1：「好提示词 vs 坏提示词」对比实验——同一任务用模糊指令和结构化指令各问一次，对比结果
- [ ] 实践 2：让 Claude 生成一个 Artifact（可分享的网页），比如把一份笔记变成可视化报告

**产出**：`notes/` 里记一篇《我的提示词心得》

### 第 2 周：AI 协作方法论（下）

**上课**：
- [ ] AI Fluency 后半 + 证书考试 🎓
- [ ] （选读）AI Capabilities and Limitations

**本机实践**：
- [ ] 实践 3：用 4D 框架完整跑一个真实任务（如"整理 github-trending-weekly 的改版思路"）
- [ ] 实践 4：盘点本机已有配置（CLAUDE.md、已装插件），让 Claude 逐一讲解它们在干什么

### 第 3 周：Claude Code 101（上）+ 命令行与 Git 入门

**上课**：
- [ ] Claude Code 101 前半：工作原理、Explore→Plan→Code→Commit 工作流、上下文管理

**加油包**：终端/命令行、Git 基础（这周必须补，后面全靠它们）

**本机实践**：
- [ ] 实践 5：完整走一遍 Explore→Plan→Code→Commit——选个小需求（如给 github-trending-weekly 加功能），Claude 先调研、再出方案、你审批后执行、最后提交版本
- [ ] 实践 6：亲手用 Git 查看历史、回滚一次改动，体会"版本控制 = 后悔药"

**关键概念（PM 版）**：
- **CLAUDE.md** = 新员工入职手册，Claude 每次"上班"先读它
- **上下文（Context）** = Claude 的工作记忆，容量有限，要学会管理
- **Commit（提交）** = 给项目存一个带说明的历史版本存档点

### 第 4 周：Claude Code 101（下）+ Cowork

**上课**：
- [ ] Claude Code 101 后半：Code Review、Subagents、Skills、MCP、Hooks + 课程测验 🎓
- [ ] Introduction to Claude Cowork

**本机实践**：
- [ ] 实践 7：让 Claude 用 Subagent 做一次大范围调研，体验"分身干活"
- [ ] 实践 8：解剖已装的 context7 插件，直观理解 MCP
- [ ] 实践 9：配一个简单的 Hook（如"每次改完文件自动做某项检查"），理解自动化审批流

### 第 5 周：Agent Skills 深入

**上课**：
- [ ] Introduction to Agent Skills 全部 6 模块

**本机实践**：
- [ ] 实践 10：解剖现有的 story-hook-analyzer skill，看真实 skill 的完整结构
- [ ] 实践 11：**从零创建你自己的 skill**（如「竞品分析 SOP」或「PRD 评审清单」），把 PM 经验固化为 Claude 的技能
- [ ] 实践 12：用 skill-creator 测试和优化你的 skill

### 第 6 周：Subagents + Claude Code in Action（第一个完整小项目）

**上课**：
- [ ] Introduction to Subagents
- [ ] Claude Code in Action

**加油包**：前端/后端概念

**本机实践**：
- [ ] 实践 13：**中期项目**——从零到一做一个完整的小工具（候选：个人仪表盘网页、小说拆解流水线、每周 GitHub 趋势自动报告）。要求走完整流程：需求 → 方案 → 开发 → 版本管理 → 复盘。你当产品负责人 + 技术决策者，Claude 当开发，但每个技术决策你都要能说出"为什么"

### 第 7–8 周：MCP 双课（接通外部世界）

**上课**：
- [ ] Introduction to MCP（完整学，Python 代码部分 Claude 陪你逐行走读）
- [ ] MCP: Advanced Topics（重点理解概念：认证、部署、生产化）

**加油包**：API 概念、JSON、Python 读代码基础

**本机实践**：
- [ ] 实践 14：跟课搭一个最小 MCP 服务器（Claude 写码你读懂），让本机 Claude Code 真的连上它用起来
- [ ] 实践 15：给中期项目接入一个实用的 MCP 能力（如联网取数）

### 第 9 周：Claude Platform 101 + API 课（上）

**上课**：
- [ ] Claude Platform 101
- [ ] Building with the Claude API 前半

**本机实践**：
- [ ] 实践 16：申请 API Key，用最简单的方式调用一次 Claude API，看懂请求和返回的每个字段——理解"你平时聊天的界面，底层就是这个"
- [ ] 实践 17：算一笔账：按 token 计费是什么意思，一个功能的成本怎么估（PM 必备）

### 第 10 周：API 课（下）+ 毕业项目

**上课**：
- [ ] Building with the Claude API 后半
- [ ] AI Fluency for Builders（轻松收尾）

**本机实践**：
- [ ] **毕业项目**：独立完成一个调用 Claude API 的真实产品雏形，综合运用 CLAUDE.md + Skills + Subagents + MCP + API。候选方向：求职材料智能流水线（升级 ai-job）、小说拆解 SaaS 雏形、你自己立项的点子。标准：你能向一个工程师讲清楚整个系统是怎么运转的

**可选加餐（第 11 周起）**：Claude with Amazon Bedrock / Google Vertex AI（若涉及企业云平台选型再学）

## 五、Skilljar 之外的学习资源

1. **官方文档**（https://code.claude.com/docs）— Claude Code 说明书，遇到具体问题时查
2. **Anthropic Academy**（https://www.anthropic.com/learn）— 课程官网入口，持续上新
3. **Anthropic 官方 YouTube** — 课程配套视频和产品演示，适合通勤看
4. **Anthropic 工程博客**（https://www.anthropic.com/engineering）— 《Claude Code Best Practices》等一手好料，可安排精读
5. **在 Claude Code 里直接问**：任何"Claude Code 能不能做 X"的问题随时问 Claude，它有专门的查证助手，比翻文档快
6. **拆解范例**：已装的 superpowers、skill-creator、context7 插件都是高质量样本，学到对应章节逐个解剖
7. **费曼输出法**：每周把所学用自己的话写进 `notes/`，讲得出来才算学会

### 点亮流程（成就地图）

1. 学完一个知识点/课程/项目后，告诉 Claude：「点亮 XX」
2. Claude 更新 `docs/data/anthropic/progress.json` 并提交推送——**每次点亮本身就是一次 GitHub 贡献**，热力图同步 +1（地图右上角可切换到 OpenAI Academy 版，其进度在 `docs/data/openai/progress.json`，点亮时说明「点亮 OpenAI 的 XX」即可）

> 热力图小贴士：GitHub 对外默认只统计公开仓库的贡献。若想让私有仓库的贡献也计入地图热力图，在 GitHub → Settings → Profile → Contributions & activity 里勾选 "Include private contributions on my profile"。
